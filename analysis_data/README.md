# analysis_data

This folder contains the utilities used to make dataset splits stable and to generate dataset reports for the segmentation project.

## Purpose

`analysis_data/` is responsible for two things:

1. creating fixed `train/val/test` manifests so experiments are reproducible
2. exporting dataset statistics and dataset summaries into `reports/`

## Architecture

```text
analysis_data/
|-- README.md
|-- generate_splits.py
|-- analyze_datasets.py
`-- reports/
    |-- current_dataset_status.md
    |-- cvc_clinicdb_summary.json
    |-- cvc_clinicdb_summary.md
    |-- kvasir_seg_summary.json
    |-- kvasir_seg_summary.md
    `-- dataset_overview.json
```

## File Roles

### `generate_splits.py`

Creates deterministic split manifests inside each dataset folder:

- `data/CVC-ClinicDB/splits/train.txt`
- `data/CVC-ClinicDB/splits/val.txt`
- `data/CVC-ClinicDB/splits/test.txt`
- `data/Kvasir-SEG/splits/train.txt`
- `data/Kvasir-SEG/splits/val.txt`
- `data/Kvasir-SEG/splits/test.txt`

### `analyze_datasets.py`

Scans the datasets and writes reports containing:

- number of image/mask pairs
- split counts
- common image sizes
- RGB mean/std
- foreground ratio
- RGB pixel histogram
- mask value presence
- example sample file paths

### `reports/`

Stores generated outputs so you can inspect dataset status without rescanning manually every time.

## Recommended Workflow

Run these commands in order:

```bash
python analysis_data/generate_splits.py --dataset all --seed 1337
python analysis_data/analyze_datasets.py --dataset all
```

If you only want one dataset:

```bash
python analysis_data/generate_splits.py --dataset cvc --seed 1337
python analysis_data/analyze_datasets.py --dataset cvc
```

```bash
python analysis_data/generate_splits.py --dataset kvasir --seed 1337
python analysis_data/analyze_datasets.py --dataset kvasir
```

## Notes

- The training and testing scripts now prefer manifests in each dataset's `splits/` folder.
- This means once you generate splits here, the rest of the project will use them consistently.
- If you want to regenerate new splits, change the seed explicitly and rerun both scripts.
